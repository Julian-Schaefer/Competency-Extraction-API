store.py¶

Allows Initialization of the Database with Competencies and provides the termStore as well as the sequenceStore.

class app.store.Store(language='de')¶

Bases: object

The Store class provides the functionality to initialize the Graph Database with EU-ESCO competencies. Furthermore, it also acts as the termStore and as the sequenceStore which are used by the app.competency_extractor.PaperCompetencyExtractor class.

Parameters:

language (str) – Language to use (defaults to German)

Variables:

preprocessor – An instance of the app.preprocessing_utils.PreprocessorGerman class to preprocess the labels of Competencies

initialize()¶

Initializes the Database with Competencies imported from a EU-ESCO compatible .csv File. The Location of the .csv file has to be specified using the environment variable “DATA_FILE”

exception app.store.StoreAlreadyInitialized¶

Bases: Exception

Raised when the Store has already been initialized.

class app.store.StoreLocal¶

Bases: object

The StoreLocal class provides the same functionality as the Store class. The only difference is that the StoreLocal class can be used locally without having to start the server and without having to connect to the Database.

Variables:

  • preprocessor – An instance of the app.preprocessing_utils.PreprocessorGerman class to preprocess the labels of Competencies

  • store_df – A DataFrame representation of a .csv file (located by the Environment Variable “LABELED_COMPETENCIES_FILE”) which contains all preferred and alternative labels that are contained in the EU ESCO API.
